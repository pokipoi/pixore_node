# Image Resize (PIL) Node

图片缩放节点 - 支持像素/百分比、stretch/fit/fill 三种缩放模式和多种插值算法。

## 功能

高效的图片缩放处理，提供多种缩放模式和插值算法选择：

- **三种缩放模式**: Stretch（拉伸）、Fit（等比+填充）、Fill（等比+裁剪）
- **灵活的尺寸单位**: 像素或百分比
- **多种插值算法**: LANCZOS、BICUBIC、BILINEAR 等
- **Fit 模式的填充选项**: 黑色、白色或透明

## 输入

| 端口 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| image | IMAGE | - | 待缩放的图片 |
| measurement | STRING | "pixels" | 尺寸单位: pixels(像素) 或 percentage(百分比) |
| width | INT | 512 | 目标宽度。设为 0 则根据高度等比计算 |
| height | INT | 512 | 目标高度。设为 0 则根据宽度等比计算 |
| fit | STRING | "fit" | 缩放模式: stretch(直接拉伸), fit(等比缩放+画布居中), fill(等比缩放+中心裁剪) |
| pad_color | STRING | "black" | Fit 模式下的填充颜色 |

## 输出

| 端口 | 类型 | 说明 |
|------|------|------|
| image | IMAGE | 缩放后的图片 |

## 缩放模式说明

### Stretch - 直接拉伸

```
原图 (200x300)  →  Stretch (512x512)
    →→→→→→→→→→→→→→→→→→→
```

特点:
- 不保持长宽比
- 可能导致图片变形
- 处理速度最快

### Fit - 等比缩放 + 填充

```
原图 (200x300)  →  Fit (512x512, black)
      ┌─────┐      ┌──────────────┐
      │     │      │   ░░░░░░░░   │
      │     │  →   │   ░┌───┐░░   │
      │     │      │   ░│   │░░   │
      └─────┘      │   ░│   │░░   │
                   │   ░└───┘░░   │
                   │   ░░░░░░░░   │
                   └──────────────┘
```

特点:
- 保持长宽比
- 周围填充指定颜色
- 适合需要固定尺寸且不损失内容的场景

### Fill - 等比缩放 + 裁剪

```
原图 (200x300)  →  Fill (512x512)
      ┌─────┐      ┌──────────────┐
      │     │      │   ┌────────┐ │
      │     │  →   │   │        │ │
      │     │      │   │  内容  │ │
      └─────┘      │   │        │ │
                   │   └────────┘ │
                   └──────────────┘
```

特点:
- 保持长宽比
- 从中心裁剪多余部分
- 适合需要固定尺寸且优先显示中心内容的场景

## 使用示例

### 等比缩放到固定尺寸（保留内容）

```json
{
  "width": 512,
  "height": 512,
  "fit": "fit",
  "pad_color": "white"
}
```

### 制作缩略图（中心裁剪）

```json
{
  "width": 256,
  "height": 256,
  "fit": "fill"
}
```

### 按百分比缩小到 50%

```json
{
  "measurement": "percentage",
  "width": 50,
  "height": 50,
  "fit": "stretch"
}
```

### 保持宽度，按比例计算高度

```json
{
  "width": 800,
  "height": 0,
  "fit": "fit"
}
```

## 插值算法

支持的插值算法（质量从高到低）：

- LANCZOS - 最高质量，处理速度最慢
- BICUBIC - 高质量
- BILINEAR - 中等质量
- NEAREST - 最快，质量最低

## 典型工作流

```
File Input → Load Image → Image Resize → Save/Output
```

## 性能考虑

- Fit 和 Fill 模式需要额外计算，比 Stretch 稍慢
- LANCZOS 插值提供最佳质量但速度最慢
- 对于大分辨率图片，优先使用 Fill 避免大面积填充

## 常见问题

**Q: Fit 和 Fill 模式有什么区别？**
A: Fit 保留完整内容并填充空白，Fill 保留固定尺寸并裁剪超出部分

**Q: 如何保持原始宽高比？**
A: 使用 Fit 或 Fill 模式，将 height 或 width 设为 0 让其自动计算

**Q: 哪种模式质量最好？**
A: Fit 和 Fill 都可以，选择取决于你的需求。质量最好用 LANCZOS 插值

**Q: 为什么缩放后的图片质量下降？**
A: 尝试使用 LANCZOS 插值算法，质量会更好
